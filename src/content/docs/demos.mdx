Seeing a method run on *your own* photo beats reading about it. Three demos so far —
one built into this page, two embedded from Hugging Face Spaces — each tied to a cell
of the [applications map](../applications-map/), each with an honest note on what it
does and doesn't show.

## In this browser: zero-shot identification (CLIP)

Pick a photo (a product, a part, a pile of scrap) and a set of candidate labels;
[CLIP](https://arxiv.org/abs/2103.00020) scores the labels against the image. It runs
**entirely in this browser tab** via
[transformers.js](https://huggingface.co/docs/transformers.js) — the photo is never
uploaded. That is the practical argument for local models on sensitive data (see
[Tools & hardware](../tools/#local--self-hosted-inference)).

<ZeroShotDemo client:only="react" />

**What it shows:** open-vocabulary recognition — the *identity* column of the
[CV taxonomy](../applications-map/cv-taxonomy/) — with labels you choose at run time.
**What it doesn't:** reliability. Scores are relative to *your* candidate list, and a
confident-looking score proves nothing about labels you didn't offer. The gap between
an impressive demo and a validated method is what the map's maturity notes track.

## Embedded: segmentation and depth

### Segment Anything (in-browser, WebGPU)

SAM segments whatever you click — the *detection & segmentation* step behind
component-counting at end-of-life. This Space also runs fully client-side.

<iframe src="https://webml-community-segment-anything-webgpu.static.hf.space/index.html"
        width="100%" height="520"
        style="border:1px solid var(--sl-color-hairline-shade);border-radius:6px"
        loading="lazy" title="Demo: Segment Anything WebGPU"></iframe>

[Open directly](https://huggingface.co/spaces/webml-community/segment-anything-webgpu)
if the embed fails to load. **What it doesn't show:** class assignment. SAM finds
regions, not component types, and that missing layer is what separates the foundation
model from a usable bill of components.

### Depth Anything V2 (hosted)

Monocular depth estimation — the *quantity/geometry* column: error-bounded geometry
from a single photo. This one calls a hosted GPU, so **your image leaves the
browser**; use test images, not confidential ones.

[Open directly](https://huggingface.co/spaces/depth-anything/Depth-Anything-V2)

reflective, transparent, and thin parts that dominate real products.

## Two ways to add a demo

- **External app (most common):** anything that *calls a hosted model* needs a
  backend, so host it separately (Hugging Face Space, an institute server, a small
  FastAPI app) and embed it with an `<iframe>`. Keep a text link too, for when the
  embed can't load.

  ```html
  <iframe src="https://example.org/your-demo/" width="100%" height="480"
          style="border:1px solid #ccc;border-radius:6px"
          loading="lazy" title="Demo: category name"></iframe>
  ```

- **In-page island:** a client-side widget ships as an Astro island, no backend —
  including real in-browser inference via transformers.js. Copy
  [`src/components/ZeroShotDemo.tsx`](https://github.com/simonvanlierde/ai4ie-demo/blob/main/src/components/ZeroShotDemo.tsx),
  then drop `<YourDemo client:only="react" />` into an `.mdx` page.

When adding a demo, link it from the matching entry in
[`src/data/applications.yaml`](https://github.com/simonvanlierde/ai4ie-demo/blob/main/src/data/applications.yaml)
(`demo:` field) so the [map](../applications-map/) points to it.
